# Build Online Shop UMKM MVP — Next.js SPA + TypeScript Backend Clean Architecture

Saya ingin kamu membuat project **Online Shop UMKM MVP** dari awal.

Project ini adalah **single-store online shop untuk UMKM**, bukan marketplace, bukan Shopify clone, dan bukan platform kompleks. Fokus utama adalah MVP yang rapi, clean, mobile-first, reusable, dan siap dikembangkan bertahap.

Saya akan menyertakan screenshot sebagai **referensi UX flow saja**. Screenshot hanya digunakan untuk memahami pola:
- product listing
- product detail
- cart drawer
- checkout
- order summary
- payment/order confirmation

PENTING:
- Jangan meniru persis UI dari screenshot.
- Jangan meniru warna dari screenshot.
- Jangan meniru font dari screenshot.
- Jangan meniru logo, nama brand, layout detail, icon style, asset, shape, spacing, atau visual identity dari screenshot.
- Buat desain original yang clean, simple, modern, profesional, mobile-first, dan cocok untuk UMKM.
- Jangan terlalu warna-warni.
- Gunakan primary color secukupnya.
- UI harus terasa modern dan profesional, bukan vibe coding yang norak.

---

# 1. Product Goal

Bangun MVP online shop UMKM dengan flow:

1. Customer membuka storefront.
2. Customer melihat daftar produk.
3. Customer membuka detail produk.
4. Customer add to cart.
5. Cart drawer muncul.
6. Customer bisa update quantity atau remove item.
7. Customer checkout.
8. Customer mengisi contact dan alamat.
9. Customer memilih shipping manual.
10. Customer memilih payment manual fake QRIS.
11. Backend membuat order.
12. Customer melihat halaman order confirmation / tracking sederhana.
13. Customer melihat instruksi pembayaran fake QRIS.

MVP ini harus sederhana tapi arsitekturnya rapi supaya nanti bisa dikembangkan menjadi:
- management dashboard
- product CRUD
- order management
- payment gateway real
- shipping integration
- multi-store
- theme setting
- analytics basic

Untuk sekarang, **management app cukup dibuat folder/app placeholder saja**.

# 2. Main Requirements

Gunakan stack:

- Monorepo: pnpm + Turborepo
- Frontend shop: Next.js + TypeScript + Tailwind CSS
- Frontend behavior: SPA-like experience
- Backend: Node.js + Fastify + TypeScript
- Database: PostgreSQL + Drizzle ORM
- Validation: Zod
- Cart state: Zustand + localStorage
- API client: package reusable
- Shared types: package reusable

Payment:
- Manual fake QRIS only
- Tidak ada payment gateway real
- Tidak ada Midtrans/Xendit/Duitku/Tripay dulu

Shipping:
- Manual shipping only
- Flat rate dari database
- Tidak ada RajaOngkir/Biteship dulu

Management:
- Buat folder/app `apps/management`
- Placeholder page only
- Harus bisa reuse component dari `packages/ui`

# 7. Backend Architecture

Path:

    backend

Gunakan Fastify + TypeScript.

Architecture:
- Clean Architecture
- DDD ringan
- Separation of Concern
- Ports and Adapters

Setiap module penting harus punya struktur seperti:

    backend/src/modules/order/
    ├─ domain/
    │  ├─ order.entity.ts
    │  ├─ order-status.ts
    │  └─ order.errors.ts
    │
    ├─ application/
    │  ├─ create-order.use-case.ts
    │  ├─ get-order-by-code.use-case.ts
    │  └─ ports/
    │     ├─ order-repository.port.ts
    │     ├─ catalog-repository.port.ts
    │     ├─ shipping-repository.port.ts
    │     └─ payment-service.port.ts
    │
    ├─ infrastructure/
    │  ├─ drizzle-order.repository.ts
    │  └─ order.mapper.ts
    │
    ├─ presentation/
    │  ├─ order.routes.ts
    │  └─ order.controller.ts
    │
    └─ order.module.ts

Rules:
- Domain tidak boleh import Fastify.
- Domain tidak boleh import Drizzle.
- Domain tidak boleh import database.
- Domain tidak boleh import HTTP.
- Application berisi use-case dan port interface.
- Infrastructure implement repository dan database query.
- Presentation hanya route/controller HTTP.
- Controller tidak boleh berisi business logic.
- Controller tidak boleh query database langsung.
- Use-case tidak boleh tergantung Fastify.
- Validasi request pakai Zod di boundary.
- Error handling konsisten.
- Jangan over-engineer.
- Jangan bikin microservice.
- Jangan bikin abstraction berlebihan untuk fitur kecil.

    stores
    products
    product_images

    discounts

    order_items
    order_addresses

General rules:
- Gunakan uuid primary key.
- Gunakan created_at dan updated_at jika relevan.

# 11. API Response Format

    {

# 19. Backend Business Rules

Product:
- Product inactive tidak boleh tampil.
- Product inactive tidak boleh checkout.
- Quantity minimal 1.
- Quantity tidak boleh lebih dari stock.
- Stock 0 berarti unavailable.

Checkout:
- Backend wajib recalculate all totals.
- Backend wajib validate product from database.
- Backend wajib validate shipping method from database.
- Backend wajib validate discount from database.
- Backend wajib create order item snapshot.
- Backend wajib create payment instruction.
- Backend wajib return order code.

Order:
- Order status awal `pending_payment`.
- Order code harus unique.
- Order detail bisa diakses by order code.

Payment:
- Payment method hanya `manual_fake_qris`.
- Payment status awal `pending`.
- Payment confirmation set status `reviewing`.
- Tidak ada real payment processing.

Shipping:
- Manual flat rate.
- Shipping amount dari database.
- Tidak ada kurir real.

Discount:
- Code `HEMAT10` harus bekerja.
- Percentage discount dihitung dari subtotal product.
- Discount tidak boleh membuat total negatif.
- Invalid discount harus return error jelas atau ignored dengan message jelas. Pilih salah satu yang konsisten.
